**1. Add retraction logic to `sync_event_for_household` (VIS-04)**

In `sync_event_for_household`, AFTER the existing loop that syncs to recipients AND after the `google_sync_at` update block, add retraction logic for non-recipients.

The key insight: `_sync_recipients` returns only the owner for private events, but the event may still exist on the partner's GCal from when it was shared. We need to delete it.

After the line `users = self._sync_recipients(event)` (line ~330), add:
```python
        all_household = self._household_users(event.calendar_id)
        recipient_ids = {u.id for u in users}
        non_recipients = [u for u in all_household if u.id not in recipient_ids]
```

After the existing recipient sync loop ends and before the `google_sync_at` update, add a retraction loop:
```python
        # Retract from non-recipients (shared→private removes from partner's GCal)
        if not deleted:
            for user in non_recipients:
                try:
                    creds = self._credentials_for_user(user)
                    if not creds:
                        continue
                    service = self._google_service(creds)
                    cal_id = self._ensure_user_calendar(service, user)
                    existing = self._find_google_event(service, cal_id, event.id)
                    if existing:
                        service.events().delete(calendarId=cal_id, eventId=existing["id"]).execute()
                        synced_events += 1
                except Exception as exc:
                    errors.append(f"{user.email}: {exc}")
```

This handles:
- shared→private: recipients=owner, non_recipients=partner → partner's copy deleted ✅
- private→shared: recipients=all, non_recipients=empty → no retraction, event pushed to all ✅
- delete: `deleted=True` skips retraction block (delete already handled in recipient loop) ✅

If GCal delete fails, log warning and continue per user decision (exception is caught and added to errors list).

```python
def test_sync_retraction_deletes_from_partner(authenticated_client, monkeypatch):
    """When visibility changes to private, sync should retract event from partner's GCal."""
    calls = {"sync_to": [], "delete_from": []}

    class FakeResult:
        users_synced = 1
        events_synced = 1
        errors = []

    def fake_sync(self, event, deleted=False):
        calls["sync_to"].append({"visibility": getattr(event, "visibility", "shared"), "deleted": deleted})
        return FakeResult()

    import app.sync.service
    monkeypatch.setattr(app.sync.service.GoogleSyncService, "sync_event_for_household", fake_sync)

    # Create shared event, then update to private
    create_resp = authenticated_client.post(
        "/api/events",
        json={
            "title": "Was Shared",
            "start_at": "2026-03-18T10:00:00",
            "end_at": "2026-03-18T11:00:00",
            "timezone": "UTC",
            "visibility": "shared",
        },
    )
    assert create_resp.status_code == 201
    event_id = create_resp.json()["id"]

    update_resp = authenticated_client.put(
        f"/api/events/{event_id}",
        json={"visibility": "private"},
    )
    assert update_resp.status_code == 200
    assert update_resp.json()["visibility"] == "private"

    # sync_event_for_household called on both create and update
    assert len(calls["sync_to"]) >= 2
    assert calls["sync_to"][-1]["visibility"] == "private"
```
