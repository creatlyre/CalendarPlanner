   - In views that need plan info (base.html uses it), pass `user_plan` in context.
   - For existing views that don't need gating, plan info in base.html can be loaded via a lightweight approach: add `user_plan` to the context dict in views that extend base.html.

   - Link to `/billing/settings`.
