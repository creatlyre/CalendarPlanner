3. Create Supabase migration `supabase/migrations/20260323_billing.sql`:
   - `subscriptions` table: `id` UUID PK, `user_id` UUID FK to auth.users (UNIQUE), `stripe_customer_id` TEXT UNIQUE, `stripe_subscription_id` TEXT UNIQUE, `plan` TEXT NOT NULL DEFAULT 'free' CHECK IN ('free','pro','family_plus'), `status` TEXT NOT NULL DEFAULT 'active' CHECK IN ('active','past_due','canceled','incomplete'), `current_period_end` TIMESTAMPTZ, `cancel_at_period_end` BOOLEAN DEFAULT false, `created_at` TIMESTAMPTZ DEFAULT now(), `updated_at` TIMESTAMPTZ DEFAULT now().
   - `billing_events` table: `id` UUID PK, `user_id` UUID FK to auth.users, `event_type` TEXT NOT NULL CHECK IN ('signup','subscribe','plan_change','cancel','churn','payment_failed'), `plan` TEXT, `stripe_event_id` TEXT UNIQUE, `metadata` JSONB DEFAULT '{}', `created_at` TIMESTAMPTZ DEFAULT now().
   - Indexes on subscriptions(user_id), subscriptions(stripe_customer_id), billing_events(user_id, created_at DESC), billing_events(stripe_event_id).
   - RLS enabled on both tables. Policies: users can SELECT own rows; service role can INSERT/UPDATE.

<task type="auto">
  <name>Task 2: Stripe checkout and webhook endpoints</name>
  <files>app/billing/service.py, app/billing/routes.py, main.py</files>
  <action>
1. Create `app/billing/service.py` — `BillingService(repo: BillingRepository)`:
   - `create_checkout_session(user_id, email, plan, success_url, cancel_url)`:
     - Get or create Stripe customer (by email, store stripe_customer_id via repo).
     - Map plan to Stripe Price ID from env vars (STRIPE_PRO_PRICE_ID, STRIPE_FAMILY_PLUS_PRICE_ID — add to config.py).
     - Call `stripe.checkout.Session.create(mode="subscription", customer=cust_id, line_items=[{price: price_id, quantity: 1}], success_url, cancel_url, metadata={"user_id": user_id, "plan": plan})`.
     - Return session URL.
   - `create_portal_session(stripe_customer_id, return_url)`:
     - Call `stripe.billing_portal.Session.create(customer=stripe_customer_id, return_url=return_url)`.
     - Return portal URL.
   - `handle_webhook_event(payload, sig_header)`:
     - Verify signature with `stripe.Webhook.construct_event(payload, sig_header, settings.STRIPE_WEBHOOK_SECRET)`.
     - Handle event types:
       - `checkout.session.completed` → upsert_subscription with plan from metadata, log 'subscribe' event.
       - `customer.subscription.updated` → update plan/status/period_end, log 'plan_change' if plan changed.
       - `customer.subscription.deleted` → update status to 'canceled', log 'churn' event.
       - `invoice.payment_failed` → update status to 'past_due', log 'payment_failed' event.
     - Extract user_id from subscription metadata or lookup by stripe_customer_id.

2. Create `app/billing/routes.py` — FastAPI router with prefix `/api/billing`:
   - `POST /checkout` — Requires auth (get_current_user). Accepts CheckoutRequest body. Returns redirect URL or JSON with checkout URL. Uses BillingService.create_checkout_session.
   - `POST /webhook` — NO auth required (Stripe calls this). Reads raw body + Stripe-Signature header. Calls BillingService.handle_webhook_event. Returns 200 OK. On verification failure returns 400.
   - `POST /portal` — Requires auth. Looks up user's stripe_customer_id, creates portal session, returns redirect URL.

3. Register in `main.py`:
   - Import `from app.billing.routes import router as billing_router`
   - Add `app.include_router(billing_router)`
   - Add `/api/billing/webhook` to `SessionValidationMiddleware` public_routes set.

4. Add `STRIPE_PRO_PRICE_ID: str = ""` and `STRIPE_FAMILY_PLUS_PRICE_ID: str = ""` to config.py Settings.
