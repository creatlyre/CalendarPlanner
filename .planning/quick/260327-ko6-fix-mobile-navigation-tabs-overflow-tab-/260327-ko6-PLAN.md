# Plan: Fix Mobile Navigation Tabs Overflow

## Task 1: Raise desktop nav breakpoints in base.html

**files:** `app/templates/base.html`
**action:**
1. Change all 6 nav links (lines 31-39) from `hidden md:inline-flex` to `hidden lg:inline-flex`
2. Change Quick Add button (line 47) from `hidden md:inline-flex` to `hidden lg:inline-flex`
3. Change FAB button (line 195) from `md:hidden` to `lg:hidden`

**verify:** Grep for `md:inline-flex` in base.html — should be 0 nav link occurrences. Grep for `lg:inline-flex` — should find all nav links.
**done:** All desktop nav links and Quick Add button use `lg:inline-flex` breakpoint

## Task 2: Update CSS media queries in input.css

**files:** `public/css/input.css`
**action:**
1. Change `@media (max-width: 767px)` (line 817) to `@media (max-width: 1023px)` so mobile-compact styles apply to tablets too
2. Change `@media (min-width: 768px) { .mobile-bottom-nav { display: none; } }` (line 1171) to `@media (min-width: 1024px)` so bottom nav stays visible on tablets

**verify:** Check input.css has no `767px` or `768px` nav-related breakpoints remaining
**done:** Mobile styling and bottom nav cover all screens below 1024px

## Task 3: Rebuild Tailwind CSS

**files:** `public/css/style.css`
**action:** Run `npx @tailwindcss/cli -i public/css/input.css -o public/css/style.css --minify`
**verify:** style.css contains `min-width:1024px` for `.mobile-bottom-nav` hide rule and `max-width:1023px` for mobile overrides
**done:** Compiled CSS reflects updated breakpoints
